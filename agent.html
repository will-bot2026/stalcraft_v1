UltimateBuild Agent Onboarding

Browser-readable setup notes for OpenClaw, Hermes, and other coding agents working on the UltimateBuild STALCRAFT calculator and optimizer.

Fast Path

From the repository root, run the idempotent quickstart script:

scripts/agent-quickstart.sh

To start the local web server after setup:

scripts/agent-quickstart.sh --dev

The SvelteKit dev server binds to 127.0.0.1. Vite prints the final port; the default is usually http://127.0.0.1:5173.

Repository Purpose

UltimateBuild calculates and optimizes STALCRAFT artifact builds. It combines normalized artifact/container/stat data, Wiki-parity stat calculation, prompt/objective parsing, market-aware price snapshots, and a SvelteKit UI.

Preserve calculator and optimizer behavior unless the task explicitly requires a change. Candidate search can be optimized, but final display should be based on exact calculator evaluation.

Install And Run Locally

  1. Open a shell at the repository root.
  2. Run scripts/agent-quickstart.sh.
  3. If checks pass, start the web app with scripts/agent-quickstart.sh --dev or pnpm --filter @ultimatebuild/web dev.
  4. For CLI smoke testing, run pnpm optimize -- "best movement speed under 0.5 radiation".
pnpm install --frozen-lockfile
pnpm web:check
pnpm test
pnpm typecheck
pnpm --filter @ultimatebuild/web dev

How The Code Connects

Data

packages/stalcraft-data reads normalized catalogs from data/normalized and exposes artifact, container, and stat metadata.

Calculator

packages/stalcraft-core calculates raw artifact stats and final build stats after container effectiveness and protection.

Optimizer

packages/stalcraft-optimizer generates legal candidates, prunes search space, scores objectives, and returns verified builds.

Web

apps/web/src/routes/+page.server.ts runs server-side optimization and market snapshot loading. +page.svelte renders the UI.

Market

packages/stalcraft-market handles local SQLite market import/export. The deployed web app should read snapshots, not write SQLite.

CLI

apps/cli is the terminal entry point for optimizer prompts.

Vercel Setup

Use Vercel for the SvelteKit web app and static/read-only market data. Do not run long STALCRAFT market pulls or SQLite writes inside Vercel functions.

  • Root directory: repository root
  • Install command: pnpm install --frozen-lockfile
  • Build command: pnpm --filter @ultimatebuild/web build
  • Framework preset: SvelteKit
  • Node version: use a current Vercel-supported Node runtime; use Node 22+ for local full-suite market verification.

The current static snapshot path is apps/web/static/market/latest-NA.json. If a hosted snapshot URL is introduced, use named environment variables only, such as PUBLIC_MARKET_SNAPSHOT_URL for public data or MARKET_SNAPSHOT_URL for server-only reads.

Supabase Guidance

Supabase is not required by the current implementation. If a future agent adds it, treat it as hosted infrastructure for market snapshots, admin metadata, or durable pull records.

  • Do not put real Supabase URLs or keys in committed files unless they are intentionally public project identifiers.
  • Keep service-role keys server-only in Vercel or worker environment variables.
  • Expose only anon keys that are designed for browser use, and pair them with strict Row Level Security.
  • Document schema and migration steps before replacing the existing SQLite snapshot workflow.

Market Snapshot Workflow

Live market pulls require STALCRAFT API credentials in the worker environment. Name variables, but never commit values.

STALCRAFT_CLIENT_ID=...
STALCRAFT_CLIENT_SECRET=...

Typical local worker flow:

pnpm market:pull -- --db .cache/market.sqlite --region NA --items item_a,item_b --additional --days 30
pnpm market:export-snapshot -- --db .cache/market.sqlite --region NA --out apps/web/static/market/latest-NA.json
